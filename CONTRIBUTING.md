# Contributing

## Report Errors or Suggestions

1. Go to [Issues](../../issues)
2. Click "New Issue"
3. Describe the problem or suggestion clearly

We review all issues and respond promptly.

---

## Add Content

To contribute new sections or improvements:

1. Fork this repository
2. Make your changes in Markdown
3. Submit a Pull Request with a clear description

---

## Writing Guidelines

- Use clear, accessible language
- Define technical terms when first introduced
- Include German terms [in brackets] only where genuinely helpful
- Cite sources in APA format
- Add practical examples where possible

---

## Recognition

Contributors are acknowledged in the "Revised by" sections within chapters. Your GitHub username will be visible in commit history.

For privacy, we don't list email addresses publicly - contributors are credited by name or GitHub username only.

---

## Questions

Open an [issue](../../issues) to ask questions about contributing.
